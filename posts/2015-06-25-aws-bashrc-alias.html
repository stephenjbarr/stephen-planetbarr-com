If you use AWS, you have probably run into the AWS CLI which gives the user the ability to control the various web services from the command line. When using AWS CLI, it is nice to have different profiles for different projects, clients, and other logical separations that you may want to enfroce. The way to do this is with Named Profiles, which allow you to have different access keys and use different regions. To use the profile, you need to to append --profile myprofilename to the end of each aws command. This gets tedious, and a good fix for this is bash aliases.

In your .bashrc file (at the root of your home directory in Linux and probably OSX), you can add the following: 

alias awsth='aws --profile=th'
alias awslp='aws --profile=sjblaptop'
alias awsai='aws --profile=sjbai1'

Then, rather than using aws --profile=sjblaptop s3 cp..., I can use awslp s3 ....
